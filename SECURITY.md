# Security Policy

## Supported Versions
The project is supported on the following versions: **latest version** and **previous version**. Please consult the repository for updates.

## Reporting Vulnerabilities
If you discover a vulnerability, please report it by sending an email to [security@yourdomain.com](mailto:security@yourdomain.com). You can also refer to our GitHub security advisories page: [GitHub Security Advisories](LINK_TO_ADVISORY).

## What to Include in a Report
When reporting a vulnerability, please include the following information:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested resolution (if applicable)

## Response Timelines
We aim to respond to all vulnerability reports as follows:
- Acknowledgement of receipt within 72 hours
- Initial assessment of the reported vulnerability within 7 days
- Communication of findings and potential remediation steps.

## Disclosure Process
While we strive to respond to reports promptly, we request that you do not publicly disclose any vulnerability until we have had the opportunity to assess and respond.

## Safe Harbor
We offer safe harbor for security researchers who disclose vulnerabilities in good faith. Any information shared with us will be treated confidentially and will not result in legal action.

## Guidelines for Responsible Testing
We encourage ethical testing in our environment. Do not exploit any vulnerabilities you discover. Testing should be non-disruptive and respectful of user data and privacy. 

Thank you for helping us improve the security of our project!
